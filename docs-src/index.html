Aejis CSS v2.0.0.alpha2

About

Aejis CSS is an opinionated CSS scaffold for user interfaces. It based on the idea of so-called isolated components, so instead of something like this: 

<div class="panel panel-default">
  <div class="panel-heading">Hello world</div>
  <div class="panel-body">
    <p>Lorem ipsum</p>
  </div>
  <div class="panel-footer">Footnotes</div>
</div>

you write this:

<div class="group">
  <header>Hello world</header>
  <div class="body">
    <p>Lorem ipsum</p>
  </div>
  <footer>Footnotes</footer>
</div>

Installation


Aejis CSS designed to be used as SASS library, so just checkout latest version to the /vendor directory or something like that. 

@import "vendor/aejis/main";

$ae-namespace: 'cms';

Normalize.css

Normalize CSS is required to work properly. If you still don't use it — go get it and enjoy.

Autoprefixer

Autoprefixer is highly recommended to handle vendor prefixes.

Usage

Namespace

All destructive rules (styles for standard html elements, like buttons) will only work after adding namespace class to parent element. Default namespace is aejis

For example, this button will be styled:

<div class="aejis">
  <button>Press me</button>
</div>
…but this one will not: 
<div>
  <button>Press me</button>
</div>
Namespace can be changed by defining $ae-namespace variable: 
$ae-namespace: 'cms';
